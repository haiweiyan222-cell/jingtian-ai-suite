## 1. Why — 为什么要做这个系统

传统企业管理中存在系统分散、数据孤岛、组织结构混乱等结构性问题，直接影响业务流转效率与运营成本。

### 系统分散，操作割裂

企业内部同时运行景云销、2号人事、OA（蓝凌）、BI报表等多个系统，员工需要在多个系统间切换登录，数据无法跨系统流转，操作效率低下。一个审批流程可能涉及三个系统，信息在系统边界处断裂。

### 数据不共享，决策滞后

各业务系统数据独立存储，客户、产品、组织人员等基础数据在多系统中重复维护且不一致。管理层无法基于统一数据视图做决策，数据汇总依赖人工导出拼接，周期长、易出错。

### 权限体系碎片化

不同系统各自维护权限，缺乏统一的组织、角色、数据权限管理。人员调岗后需在多个系统分别调整权限，遗漏风险高。字段级、按钮级权限管控缺失，敏感数据暴露面大。

### 扩展性不足，重复建设

每个新业务需求都从零搭建，组织人员、登录鉴权、消息通知等公共能力被反复实现。缺乏可复用的中间层，研发资源浪费在重复造轮子上，新业务上线周期长。

## 2. What — 系统做什么

建设一套统一中台系统，提供单点登录入口，内部融合全公司业务，通过权限体系控制可见范围，支持灵活扩展。系统覆盖 Web 端和 APP 端，并集成外部系统。

### 2.1 统一认证与登录

- 多区号登录（+86 / +852 / +853），支持账号密码与动态验证码登录 `[Web+App]`
- 忘记密码短信验证流程，验证码有效期与频次控制 `[Web+App]`
- 首次登录强制修改密码，设备绑定与强制清除 `[Web+App]`
- 账号字段变更触发强制退出登录 `[Web]`
- 内部识别码（admin 账号专属） `[Web]`

### 2.2 工作台

- 最近使用应用（最近7个，时间倒序） `[Web+App]`
- 流程管理入口：待我审批 / 我发起的 / 我已审批 / 抄送我的 / 全部审批 `[Web+App]`
- 我的应用：添加、搜索、拖拽排序 `[Web+App]`
- 公告与消息展示，小铃铛未读提醒 `[Web+App]`
- 个人中心：基本信息、修改密码、切换岗位、退出登录 `[Web+App]`
- 下载列表管理（异步下载、导入导出文件） `[Web]`

### 2.3 基础资料

- 人员管理：增删改查、导入导出、启停用、重置密码、收货地址 `[Web]`
- 角色管理：多级角色、启停用、关联人员、数据权限绑定 `[Web]`
- 组织管理：树形组织结构、多应用体系隔离、导入导出 `[Web]`
- 客户管理、工厂管理、仓库管理 `[Web]`
- 产品管理与产品价目表 `[Web]`
- 强制清除设备绑定、登录历史记录 `[Web]`

### 2.4 消息与流程管理

- 消息中心：通知消息、审批消息、批量已读、消息跳转 `[Web+App]`
- 通知公告中心：集团推文、置顶公告、详情查看 `[Web+App]`
- 审批流程：待我审批 / 我发起的 / 我已审批 / 抄送我的 / 全部审批 `[Web+App]`
- 流程配置：审批节点配置、流程字段配置 `[Web]`
- 短信通知：羚羊主通道 + 阿里兜底 `[Web]`

### 2.5 系统设置与工具

- 操作日志、系统日志（日志中心） `[Web]`
- 字典管理、版本更新管理、前端日志列表 `[Web]`
- 消息配置、变量字段配置、错误码查询 `[Web]`
- Web 菜单管理、APP 菜单配置（超级管理员） `[Web]`
- 数据权限管理：范围定义与分配 `[Web]`

### 2.6 AI智能体与业务扩展

- AI文本对话（DeepSeek / 通义千问，深度思考 + 联网搜索） `[Web+App]`
- AI图像生成（豆包 / ChatGPT，多风格多比例） `[Web+App]`
- 总裁意见箱（含进度管控） `[Web+App]`
- 问卷调查 `[Web+App]`
- BI报表嵌入、需求收集表（飞书文档集成） `[Web+App]`
- 应用停服支持、考勤打卡 `[App]`

## 3. Impact — 预期影响

系统上线后，从效率、成本、安全、扩展性四个维度带来可量化的改善。

| 维度 | 指标 | 说明 |
|------|------|------|
| 统一入口 | 1 | 单点登录覆盖全部业务系统 |
| 系统集成 | 6+ | 景云销 / 2号人事 / OA / BI / 飞书 / 金蝶 |
| 权限管控 | 3级 | 菜单级 + 按钮级 + 字段级 |
| 双端覆盖 | 2端 | Web端 + APP端同步 |

### 效率提升

员工无需在多系统间切换，工作台一站式访问全部应用。审批流程统一入口，消息通知实时触达，减少信息查找和等待时间。基础数据一次维护、多系统共享，消除重复录入。

### 成本降低

可复用的中台中间件（登录鉴权、组织人员、消息通知、权限体系）避免新业务重复建设。统一维护降低运维成本，数据一致减少纠错成本。

### 安全增强

统一认证集中管理账号与权限，设备绑定防止账号盗用，字段级权限保护敏感数据。操作日志全量记录，满足审计合规要求。密码策略与登录频控降低暴力破解风险。

### 扩展能力

中台架构提供可扩展的基础能力层，新业务模块可基于已有组织、权限、消息能力快速搭建。应用体系隔离机制支持多业务线并行，菜单与权限配置化接入新模块。

## 5. 后续迭代计划

基于当前 V26.8 版本，后续迭代按以下方向推进。

### 近期 · V27 — AI能力深化与数据治理

AI智能体扩展更多业务场景（智能审批助手、数据问答）；金蝶数据清洗完成上线；完善字段级权限覆盖率至全模块；日志中心增加智能异常检测。

### 中期 · V28 — 我的图表与 BI 自助分析

上线"我的图表"功能，支持用户自助配置 BI 看板；打通飞书文档与中台数据的双向同步；总裁意见箱增加数据分析报告自动生成。

### 中期 · V29 — 全模块导出与数据导出中心

补齐人员管理、组织管理等模块的导出功能；建设统一数据导出中心，支持跨模块数据联合导出与定时任务。

### 远期 · V30+ — 中台能力开放与生态扩展

开放中台 API 能力（组织、权限、消息）供外部系统调用；接入更多外部系统（ERP、WMS）；探索 AI Agent 自动化项目管理场景。
